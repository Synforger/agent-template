# todos/（横断プール）

## このフォルダは何か

エージェント が管理する**横断的な開発タスクの直下プール**（2026-05-31 改定）。

「何をやるか」を1ファイル単位で管理する場所。**特定プロジェクトに属さない**タスクをここに置く。

## 何を入れる

- **横断的なテーマ**のタスク束（複数プロジェクトに跨る、特定プロジェクトに属さない）
- リファクタリング・運用改善・整理タスク
- 単発の調査タスク・検証タスク・リマインダー
- 1ファイルの中で「やること・メモ・調査結果へのリンク」をまとめて持つ

調査結果のような「育てる前提のドキュメント」は `research/` に切り出して、todosのメモ欄からリンクで参照する。

## ここに置かないもの（2026-05-31 改定）

- **特定プロジェクトに属するタスク** → `projects/<project>/todos/` (= プロジェクトを 1 か所開けばそのタスクが揃う構造)
- **repo に残す計画書・設計ドキュメント** → 対象 repo の `docs/` 配下（user の VS Code workspace と整合させるため）。ここ（エージェントの todos）は user 自身が管理したい「次アクション・進捗」のレイヤー

## 命名規則

- **トピック単位ファイル**: `kebab-case-name.md`
- 例: `rss-llm-agent-next.md`, `voice-agent-research.md`
- 1ファイル = 1テーマ。肥大化したら分割を検討

## 書き方

- `_template.md` をコピーして使う
- フロントマター: `title`, `priority`（高/通常/低）, `created`, `deadline`（任意）, `status`（open/in-progress/done）
- 本文構成: `### やること` → `### メモ`
- タスクはチェックボックス `- [ ]` / `- [x]`。完了したら `[x]` に変えて履歴として残す（削除しない）
- 大きなタスクは「完了済み（YYYY-MM-DD セッションN）」セクションを別途設けて整理してもよい

## ライフサイクル

- **読む**: `todos/` 直下のファイル一覧と中身を確認する。何が未着手か・何が進行中か・何が優先かを把握する読み方
- **作成**: 新しい取り組み・タスクが立ち上がったタイミングで作る
- **更新**: 進捗・完了に合わせて随時更新。後回しにしない
- **完了**: 全タスクが完了したら `status: done` に変える。**ファイルは削除しない**（履歴として残す）。`finished/` のような移動先フォルダは設けない
- **ボツ**: 取り組まないことが確定したらファイルごと削除する

## エージェントの挙動ルール

- **更新**: タスクに進捗・完了があった瞬間にその場で書き換える
- **完了検知**: ファイル内の全タスクが `[x]` になり `status: done` になったら、次セッションでも参照価値があるかを見て、なければ削除を提案する
- **長期滞留検知**: 長期間 open のままのタスクがあれば優先度の見直しを提案する
- **新規作成は明示指示時のみ**: user が「TODOに残して」「記録しておいて」等を明示した時だけ作る。空気を察しての自動作成・提案ベースの先回り作成は禁止（CLAUDE.md の行動指針参照）
