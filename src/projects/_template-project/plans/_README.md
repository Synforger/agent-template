# plans/ — {{プロジェクト名}} プロジェクト 計画書集約

## このフォルダの役割

本 mode の作業と並行で考えた**計画 / 段取り / 設計**を 1 か所に集約する場所。 「どう進めるか」 を file 単位で管理 (= todos = 「何をやるか」 と区別)。

## スコープ

### 入れて良いもの
- 本 mode に関する**計画 / レビュー / 採用方針 / 段取り**
- 着手前 / 進行中 / 反映済の計画 (= 反映済は `_archive/` へ)
- 設計選択肢 / 段階的移行プラン

### 入れないもの (= 別場所)
- 単一タスクの実行管理 → `../todos/`
- 技術知識の蓄積 → `../research/`
- 個人 secret / credential / セットアップ手順
- 横断的な計画 → `<agent-repo-root>/plans/`

## 構成

```
plans/
├── _README.md              # このファイル
├── _template.md            # 雛形
├── <未着手・進行中>.md
├── <大型計画>/             # サブフォルダ化した計画
└── _archive/               # 完了済 / 反映済 / 不採用確定
```

## 命名

- ファイル名 = `<plan-name>.md` (= snake-case or kebab-case)
- 複数 file で纏まる計画 = `<plan-name>/` サブフォルダ + 配下に `_README.md` + 計画書群

## 改訂運用

- 自発で改訂・追加・archive 可
- 完了済は速やかに `_archive/` 移送
- 古い計画 (= 3 ヶ月以上 status 変化なし) は 自発で user に 1 行打診
